
### 3.2 创建 CONTRIBUTING.md

创建 `CONTRIBUTING.md` 文件：

```markdown
# 贡献指南

感谢您对这个项目的兴趣！

## 如何贡献

### 报告 Bug

1. 检查 [Issues](../../issues) 中是否已有相同的 Bug 报告
2. 如果没有，创建一个新的 Issue
3. 提供以下信息：
   - Bug 的描述
   - 复现步骤
   - 预期行为
   - 实际行为
   - 截图（如果适用）

### 提交功能请求

1. 创建一个新的 Issue
2. 描述您想要的功能
3. 解释为什么这个功能有用
4. 提供可能的实现方案

### 提交 Pull Request

1. Fork 这个仓库
2. 创建一个新分支：`git checkout -b feature/amazing-feature`
3. 提交您的更改：`git commit -m 'Add some amazing feature'`
4. 推送到分支：`git push origin feature/amazing-feature`
5. 打开一个 Pull Request

## 代码风格

- 使用 4 个空格进行缩进
- 使用有意义的变量名
- 添加注释说明复杂逻辑
- 遵循 HTML、CSS、JavaScript 的最佳实践
